### 多语言

- 插件 - `i18n Ally`
- 配置，src 目录下新建 locales 文件夹，新建 `en.json` 、`zh-CN.json` 两个文件，作为多语言的入口
- 最关键的一步：

  - 在 `.vscode`文件夹下新建 `setting.json`，具体每个配置意义可以问 AI

    ```json
    {
      "i18n-ally.sourceLanguage": "zh-CN",
      "i18n-ally.localesPaths": ["src/locales"],
      "i18n-ally.displayLanguage": "zh-CN",
      "i18n-ally.extract.keygenStrategy": "slug"
      "i18n-ally.keystyle": "nested",
      "i18n-ally.extract.keygenStyle": "camelCase",
      "i18n-ally.enabledFrameworks": ["custom", "vue"],
      "cSpell.words": ["citsgbt", "cordova"]
    }
    ```

  - 在 `.vscode`文件夹下新建 `i18n-ally-custom-framework.yml`，具体配置可以问 AI

    ```yaml
    languageIds:
      - javascript
      - html
      - ftl
      - js

    usageMatchRegex:
      - "\\$i18nMsg\\((\\w+(?:\\.\\w+)*), (\\w+(?:\\.\\w+)*)\\)"
      # - "globalConfig\\.i18n\\['(.*?)'\\]" # globalConfig.i18n['key']

    refactorTemplates:
      - "this.$t('$1'，{ fallback: '$2' })"
    #   - "globalConfig.i18n['$1']"     # 其他框架模板

    monopoly: false # 是否独占模式，设置为 false 以支持其他框架
    ```

    完成上述步骤，即可在代码中 `$t('xxx')`、`$i18nMsg('xxx')`中看到被插件替换掉的文本

### 代码格式化

目测原项目不是在 VS Code 中开发，但是原来的代码风格，并不是主流的代码格式化风格，因此需要进行一些配置，尽量不要因为代码风格污染 SVN，并且符合 lint 的校验

- 下载 Prettier 插件，并新建 `.prettierrc`配置文件，配置如下

  ```json
  {
    "singleQuote": true,
    "trailingComma": "none",
    "tabWidth": 2,
    "endOfLine": "auto",
    "semi": false,
    "vueIndentScriptAndStyle": true
  }
  ```

- 接下来改造 `eslint.js`，继承 prettier 的配置即可

  ```javascript
  module.exports = {
    root: true,
    env: {
      node: true
    },
    extends: ['plugin:vue/essential', '@vue/standard', 'prettier'],
    rules: {
      'no-console': 'off',
      'no-debugger': process.env.NODE_ENV === 'production' ? 'error' : 'off'
    },
    parserOptions: {
      parser: 'babel-eslint'
    }
  }
  ```

- 除此之外，还有 VS Code 的代码格式化的风格，和 prettier 的格式化风格冲突，可在当前项目中的 `settings.json`中添加格式化风格方案

  ```json
  {
    "javascript.format.insertSpaceBeforeFunctionParenthesis": true,
    "javascript.format.insertSpaceAfterFunctionKeywordForAnonymousFunctions": true,
    "cSpell.words": ["citsgbt", "cordova"]
  }
  ```

#### 业务中的一些逻辑点

- 日历组件为啥可选择过去的事件，是因为需要补单，补单有可能补的是过去的单
- 价格范围中，最大值+的实现逻辑就是最大值加1，比如2000+其实就是2001，同理3000+就是3001
- 订单核对页中其实原来就有到达时间和离店时间，但是没用，后期可移除
- 到店时间的逻辑是：非担保房型 & 选择不需要担保 & 担保时间存在 & 非个人预付，这个时候才担保时间 - 1，其余均为到24:00
- 核对页面的价格明细，在国际酒店的情况下，有可能每天价格是丢失的，所以为了保持页面的统一，前端根据总价计算出每天的均价，显示在页面上
- 补单页面和非协议酒店预订页面，调用的搜索酒店的接口是不一样的。解决方案：在APP和H5的调用接口中，传入了fromHotelInputTips后进行分发
- 酒店接入的渠道里，有个另类，艺龙房型基本上都是担保的，所以有很多关于艺龙的逻辑，mappingInterfaceId为8即为房型渠道为8的艺龙渠道
- PC和APP代码中，有两个关于担保的关键变量，needGuarantee和useGuarantee，needGuarantee这个是详情里返回的是否需要担保，useGuarantee是表单中提交的用户选择的是否担保
- 预订时，会传入一个参数requestHotelType，告知预订的类型，3为补单和自订，2为正常的协议酒店，3为非协议酒店
- 提交核对订单页时，目前各个平台直接拿去的当前用户信息，直接扔给后台的，但是目前3个平台数据做不到一致，其实实际仅有userId是有效的

#### 名词解释

- isNonGDS 非协议房型，接入渠道不能是[3、8、19、20、22、31、41]的即为非协议房型
- mappingInterfaceId [int] 接入渠道
- 是否协议酒店，corporate [boolean]

#### 所用技术栈的坑

- bootstrap样式版本采用的float布局，可能和flex布局冲突，需要特别注意
- ft中注释中如果有尖括号，采用`<!---->`注释会引起编译错误，需要换成`<#-- -->`注释
- ft模板其实就是html，有些时候，可以像写html页面一样写ft，效果可能更好（币种的实现方式）
- 不支持编译es6

### 开发前的改造

背景：目前预订页面的的搜索页、列表页、详情页、预定页均为杂糅到一起的，这样做有个天坑，启动热加载时，当修改到js时，会造成页面的重新刷新
方案：将这几个页面拆分，每个页面作为一个单独的路由，此时，数据可进行缓存，热加载才不会造成页面的刷新，提升开发效率

### 步骤

- 改造文件ProductRoutes.js，增加新路由，那核对页为例

```javascript
 {
    path: '/product/book/check',
    name: 'ProductBookCheck',
    component: () => import('@/views/osp/booking/Booking.vue')
  }
```

- 增加信息核对页页面，这里要注意的时，从缓存中取出数据不能直接引用，必须采用ref赋值才能触发双向绑定

```javascript
<!-- 酒店下单核对页 -->
<script setup>
  import HotelOrderCheck from '@/views/components/book/hotelV2/HotelOrderCheck.vue'
  import { ref } from 'vue'

  const selectedHotelRoom = ref({})
  const selectedHotel = ref({})
  const hotelRoomBookingParam = ref({})
  const hotelMasterSearchParam = ref({})
  const requestHotelType = ref({})

  selectedHotelRoom.value = JSON.parse(localStorage.getItem('ops_selected_hotel_room') || '{}')
  selectedHotel.value = JSON.parse(localStorage.getItem('ops_selected_hotel') || '{}')
  hotelMasterSearchParam.value = JSON.parse(localStorage.getItem('ops_hotel_master_search_param') || '{}')
  hotelRoomBookingParam.value = JSON.parse(localStorage.getItem('ops_hotel_room_booking_param') || '{}')
  requestHotelType.value = JSON.parse(localStorage.getItem('ops_request_hotel_type') || '{}')
</script>

<template>
  <HotelOrderCheck
    :selected-hotel-room="selectedHotelRoom"
    :selected-hotel="selectedHotel"
    :hotel-master-search-param="hotelMasterSearchParam"
    :hotel-room-booking-param="hotelRoomBookingParam"
    :request-hotel-type="requestHotelType"
  />
</template>
```

- 在双击时，填入新页面需要的数据，关键字是: @row-dblclick

```javascript
// 点击预订方法链: showHotelRoomDetail，windowConfig.value.showHotelOrderCheckFlag = true

```
